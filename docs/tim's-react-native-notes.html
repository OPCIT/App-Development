React Native Notes

Setting up Development Environment

Install NodeJS, NPM, Expo-CLI, Android Studio (Solely for the Android Emulator), HAXM

Note: Expo-CLI is fussy about the version of NodeJS and NPM. I had to install N Install to manage the versions, as I was finding the NodeJS Windows installer was installing a lower version (4.26) that it was meant to and is required by NodeJS.

Android / iOS devices with Expo client app installed:
  • Expo client App: iTunes
  • Expo client App: Google Play

Testing

Testing for Android

  • Using expo-cli and the expo client app installed on an android device or the Android Emulator
  • Via https://snack.expo.io

Testing for iOS

  • Using expo-cli and the expo client app installed on an iOS device
  • Via https://snack.expo.io

Frameworks and Libraries

Expo - https://expo.io/tools
  • Provides an SDK and tools to build to Android and iOS, Manages Push Certificates, creates boilerplate
  • Limits your ability to use native libraries (there are work arounds) but this should not be an issue

React Navigation - https://reactnavigation.org/

A javascript library to handle navigation (i.e. between the different screens of the application as well as Authentication flow) this seems like the best of the available options for our use case

Authentication

If we require user authentication i.e. to display users priveleges card, React Native can communicate with Drupal via a REST API using JSON Web Token. Modules are avaliable to cover the basics of this in Drupal 8:

  • https://www.drupal.org/project/restui
  • https://www.drupal.org/project/jwt

See: Drupal 8 and React Native

Layout / Design

React Native Responsive UI - A library for dealing with different screen dimensions (i.e. If you want different layouts for different devices)
See: https://github.com/wcandillon/react-native-responsive-ui

React Native Responsive Screen - For scaling to different dimensions (i.e. If you want the same layout scaled on different devices)
See: https://www.npmjs.com/package/react-native-responsive-screen

React Native heavily uses CSS flexbox

CSS

  • Not all properties are supported; at least not yet.
  • Check the documentation for more info: View Style Props, Image Style Props, Text Style Props
  • Shorthand properties do not work well, so always prefer the specific ones (i.e. margin-bottom, margin-top, margin-left, margin-right instead of margin)
  • Not all properties support percentage values. To mention a few: margin, border-width and border-radius. And if someone tries to set a percentage value, RN will either ignore it completely or the application will crash.
  • RN provides flex support out of the box. Learn it and use whenever you can. It?s you best CSS ally!
